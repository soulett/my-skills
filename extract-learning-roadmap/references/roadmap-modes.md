# Roadmap Modes

## Paper Mode

Use when the main input is a paper, survey, or academic slide deck.

Preferred stage order:
1. Background and problem
2. Prerequisites
3. Core method
4. Training / inference / experiment design
5. Reproduction path
6. Extensions and advanced reading

## Project Mode

Use when the main input is a project description, README, technical design doc, or code explanation.

Preferred stage order:
1. Project goal and problem framing
2. Core architecture
3. Execution flow / data flow
4. Module-by-module understanding
5. Code reading path
6. Optimization and advanced topics

## Mixed Mode

Use when the project implements or extends a paper, or when the folder contains both theory and implementation materials.

Preferred stage order:
1. Problem and motivation
2. Theory and method
3. System architecture
4. Implementation path
5. Reproduction / extension
