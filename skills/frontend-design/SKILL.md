# Frontend Design

Design polished user interfaces and frontend concepts that are ready to hand off into implementation.

## What this skill does

This skill helps shape product interfaces, screen structure, interaction ideas, UI states, and presentation details. It is useful when a user wants help designing pages, components, flows, or visual direction before or during frontend implementation.

## When to use

Use this skill when the user wants to:
- design a page or screen
- define a component system or UI pattern
- explore interaction details or empty/loading/error states
- improve hierarchy, readability, and visual clarity
- translate product requirements into frontend structure

## Workflow

1. Clarify the product goal and target user.
2. Identify key tasks and priority content.
3. Lay out the page or component structure.
4. Define interaction patterns and edge states.
5. Suggest visual direction, spacing, and hierarchy.
6. Prepare implementation-ready guidance where helpful.

## Design checklist

Cover these when relevant:
- primary user goal
- information hierarchy
- navigation and flow
- empty/loading/error states
- responsive behavior
- accessibility considerations
- typography and spacing
- component consistency
- calls to action

## Output style

Prefer practical design output such as:
- page outlines
- section breakdowns
- component lists
- state definitions
- UX notes
- implementation hints for frontend teams

## Principles

- Optimize for clarity first.
- Make the most important actions obvious.
- Keep interactions predictable.
- Use visual emphasis intentionally.
- Design with real content and edge cases in mind.
