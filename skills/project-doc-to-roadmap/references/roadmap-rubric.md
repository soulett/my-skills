# Roadmap Rubric

Use this reference to keep the generated roadmap teachable, dependency-aware, and useful for project understanding plus interview prep.

## Core Design Rule

Do not reproduce the source document structure.

Rebuild the project into a learning order that follows:

1. foundations
2. task understanding
3. method prerequisites
4. core modules
5. training and evaluation
6. expression and interview use

## Module Count

Default to 12-20 modules.

Use fewer modules when the source is shallow.
Use more modules when the source has several clearly separable technical parts.

## Required Fields Per Module

Every module should contain:

- 前置依赖
- 面试高频程度
- 为什么学
- 学习重点
- 学到什么程度
- 国内知识帖
- 国内网课 / 视频
- 英文必看链接
- 对应项目内容

## Resource Selection

### Domestic resources

Prefer:

- bilibili
- 中国大学MOOC / 学堂在线
- 腾讯云开发者
- CSDN

Keep resources that are:

- clearly titled
- accessible now
- obviously relevant to the module
- suitable for the learner stage

Avoid:

- duplicate links
- broken pages
- content farms
- vague or weakly related posts

### English resource

Add one strong English source per module by default.

The English link should usually be one of:

- official course page
- official framework tutorial
- official project page
- classic paper page
- reputable educational article

Its role is to deepen or correct understanding, not to replace the domestic resources.

## Frequency Labels

Use these labels:

- 高频必学
- 高频扩展
- 理解加分

Choose based on likely interview value, not theoretical prestige.

## Output Shape

End the file with:

- 推荐学习顺序
- 学完后应能回答的问题

Do not add:

- 使用说明
- 本地优先资料

unless the user explicitly asks for them.
