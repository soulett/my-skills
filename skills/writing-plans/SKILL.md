# Writing Plans

Write clear execution plans before implementation so work is scoped, sequenced, and easier to review.

## What this skill does

This skill helps convert a rough task or initiative into a practical written plan. It is useful before building, refactoring, migrating, or coordinating larger changes. The focus is on creating a plan that is concrete enough to guide execution without over-engineering the document.

## When to use

Use this skill when the user wants to:
- break a task into steps
- decide implementation order
- identify dependencies and risks
- prepare a plan for review before coding
- align teammates on scope and sequencing

## Workflow

1. Define the goal and constraints.
2. Identify unknowns and assumptions.
3. Break the work into major phases or milestones.
4. Add dependencies, risks, and validation steps.
5. Produce a concise plan document with clear next actions.

## Plan structure

A good plan often includes:
- objective
- scope
- assumptions
- milestones or phases
- implementation steps
- dependencies
- risks
- validation / testing
- rollout or follow-up

## Writing guidance

- Prefer concrete steps over abstract advice.
- Sequence work in a way that reduces risk.
- Name the validation step, not just the implementation step.
- Keep the plan readable and reviewable.
- Call out what is intentionally out of scope.
